Analyze Repository

http://localhost:8000/analyze

Submit a public GitHub repository URL to extract its call graph and AST node data. Returns URLs to interactive HTML visualizations and raw JSON for javalang-parsed nodes.

Request application/json
http://localhost:8000/analyze
Body parameters
repoUrl * required string
Full GitHub repository URL. Must be public and contain Java source files.
Live request body
{
  "repoUrl": "https://github.com/owner/repo"
}
Code samples
cURL 
curl -X POST "http://localhost:8000/analyze" \
  -H "Content-Type: application/json" \
  -d '{"repoUrl":"https://github.com/owner/repo"}'
Python 
import requests

url  = "http://localhost:8000/analyze"
body = {"repoUrl": "https://github.com/owner/repo"}

resp = requests.post(url, json=body)
data = resp.json()

print(data["graphs"]["callgraph_json"])
print(data["graphs"]["callgraph_html"])
JavaScript (fetch) 
const resp = await fetch("http://localhost:8000/analyze", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ repoUrl: "https://github.com/owner/repo" })
});
const data = await resp.json();
console.log(data.graphs.callgraph_json);
console.log(data.graphs.callgraph_html);

Search Call Graph

        
http://localhost:8000/search

        
Search for methods/functions within a previously analyzed repository. Requires the repo to have been processed by /analyze first. Returns matched functions with their outgoing calls, callers, and ordered execution trace.

      

    


    


      
      

        

          

            Request
            application/json
          

          


            

              http://localhost:8000/search
            


            
Body parameters


            

              

                owner_name
                string
              

              
GitHub repository owner username. Defaults to Glank if omitted.

              
            


            

              

                repo_name
                string
              

              
Repository name. Defaults to Java-Games if omitted.

              
            


            

              

                query * required
                string
              

              
Method name or keyword to search for in the call graph. Matches against both short names and fully-qualified names.

              
            


            
Live request body


            

              
              
{
  "owner_name": "",
  "repo_name":  "",
  "query":      ""
}

            


            

          

        


        
        
Code samples

        

          

            cURL
            
          

          

            
curl -X POST "http://localhost:8000/search" \
  -H "Content-Type: application/json" \
  -d '{"owner_name":"","repo_name":"","query":""}'

          

        


        

          

            Python
            
          

          

            
import requests

url  = "http://localhost:8000/search"
body = {
    "owner_name": "",
    "repo_name":  "",
    "query":      ""
}

resp    = requests.post(url, json=body)
data    = resp.json()
results = data["results"]

for fn in results:
    print(fn["fullName"], "calls", fn["orderedExecution"])

          

        


        

          

            JavaScript (fetch)
            
          

          

            
const resp = await fetch("http://localhost:8000/search", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    owner_name: "",
    repo_name:  "",
    query:      ""
  })
});
const { count, results } = await resp.json();
results.forEach(fn => console.log(fn.fullName, fn.orderedExecution));
